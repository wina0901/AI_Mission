# 손상 문서 이미지 복원 프로젝트

## 프로젝트 개요

이 프로젝트는 노이즈가 포함된 손상 문서 이미지를 입력받아, 원본 문서에 최대한 가깝게 복원하는 이미지 복원 프로젝트입니다.  
주어진 noisy-clean paired 데이터셋을 활용해 다양한 딥러닝 기반 복원 모델을 실험하고, 성능을 비교한 뒤 최종 모델을 선정하여 테스트 문서를 복원하는 방식으로 진행했습니다.

단순히 하나의 모델을 학습시키는 데 그치지 않고, 데이터 구조를 먼저 분석하고, 이미지 크기 및 채널을 통일하는 전처리를 수행한 뒤, 여러 복원 모델의 구조적 차이를 비교하며 가장 적합한 모델을 선택하는 데 중점을 두었습니다.

---

## 이 프로젝트를 왜 했는가

문서 이미지 복원은 OCR, 문서 디지털화, 아카이빙, 자동 문서 처리 같은 실제 업무와 매우 밀접하게 연결되는 문제입니다.  
특히 문서 이미지에 노이즈가 많거나 훼손이 있는 경우에는 이후 단계의 인식 성능 자체가 크게 떨어질 수 있기 때문에, 사전에 문서를 얼마나 잘 복원하느냐가 전체 시스템의 품질에 큰 영향을 줍니다.

이번 프로젝트는 이러한 문제를 직접 다뤄보기 위해 진행했습니다.

- 컴퓨터 비전에서 단순 분류가 아니라 복원 문제를 직접 해결해보고 싶었습니다.
- 입력 이미지와 정답 이미지가 쌍을 이루는 supervised image restoration 문제를 경험해보고 싶었습니다.
- 이미지 전처리와 모델 구조에 따라 복원 품질이 어떻게 달라지는지 비교해보고 싶었습니다.
- 오토인코더 계열, U-Net, DnCNN처럼 서로 다른 복원 방식을 실제로 구현하고 성능을 검증하는 경험을 쌓고 싶었습니다.

즉, 이번 프로젝트는 단순한 이미지 처리 실습이 아니라, 손상된 시각 정보를 복원하는 실무형 문제를 딥러닝 기반으로 해결해보는 과정이었습니다.

---

## 문제 정의

- 주제: 손상된 문서 이미지 복원
- 입력: noisy document image
- 출력: cleaned document image
- 문제 유형: 이미지 복원 / 이미지 대 이미지 변환
- 데이터 구성:
  - train: 144장
  - train_cleaned: 144장
  - test: 72장

train과 train_cleaned는 noisy-clean 쌍으로 구성되어 있었고, test 데이터에는 정답 이미지가 없기 때문에 모델 간 비교를 통해 최종 모델을 선정하는 방식으로 진행했습니다.

---

## 데이터 이해 및 EDA

프로젝트 초반에는 먼저 데이터 자체를 이해하는 데 집중했습니다.

확인한 내용은 다음과 같습니다.

- train, train_cleaned, test 이미지 샘플 시각화
- 데이터 개수 확인
- 이미지 채널 수 확인
- 이미지 크기 분포 확인

EDA 결과를 통해 다음 사실을 확인했습니다.

- 모든 이미지는 3채널 RGB 형식
- train과 train_cleaned는 1:1로 대응되는 paired dataset
- test는 복원 대상만 존재
- 이미지 크기는 두 종류로 구성
  - 258 x 540
  - 420 x 540

즉, 모델 학습 전에 크기를 통일하는 전처리가 반드시 필요하다는 점을 먼저 확인할 수 있었습니다.

---

## 전처리 과정

데이터 특성을 반영하여 다음과 같은 전처리를 수행했습니다.

### 1. 이미지 크기 통일

이미지 높이가 서로 달랐기 때문에, 작은 이미지인 258 x 540을 기준 크기인 420 x 540에 맞추기 위해 패딩을 적용했습니다.  
이를 통해 모든 입력이 동일한 크기로 모델에 들어가도록 정리했습니다.

### 2. Bilateral Filter 적용

입력 noisy 이미지에는 bilateral filter를 적용했습니다.  
이 필터는 단순 블러와 달리 경계를 비교적 보존하면서 노이즈를 줄이는 데 유리하므로, 문서의 글자 윤곽을 유지하면서 잡음을 줄이기 위한 사전 처리로 사용했습니다.

### 3. Grayscale 변환

입력 이미지와 타깃 이미지 모두 grayscale로 변환했습니다.  
문서 복원에서는 색상 정보보다 글자와 배경의 대비가 더 중요하다고 판단했고, 채널 수를 줄여 모델이 복원 자체에 집중하도록 설계했습니다.

### 4. Tensor 변환 및 정규화

PyTorch 학습을 위해 이미지를 tensor 형태로 변환하고, float32 스케일링을 적용했습니다.

이 전처리 단계는 단순한 형식 변환이 아니라, 문서 복원 문제에 적합한 입력 형태를 설계하는 과정이었습니다.

## 모델링 전략

이번 프로젝트에서는 서로 다른 구조를 가진 4개의 복원 모델을 비교했습니다.

### 1. DAE

기본적인 Denoising Autoencoder 구조를 사용해 noisy 이미지를 latent representation으로 압축한 뒤 다시 복원했습니다.  
복원 문제의 가장 기본적인 기준 모델로 활용했습니다.

### 2. Multi-Scale Autoencoder

단일 스케일보다 다양한 해상도 수준의 특징을 반영할 수 있도록 멀티스케일 구조를 적용했습니다.  
문서 이미지의 큰 배경 패턴과 세부 글자 특징을 함께 학습하려는 목적이었습니다.

### 3. U-Net

인코더-디코더 구조에 skip connection을 결합한 U-Net을 적용했습니다.  
복원 문제에서는 저수준 디테일 보존이 중요한데, U-Net은 인코더의 정보를 디코더에 직접 전달할 수 있어 문서 윤곽과 글자 형태를 더 잘 복원할 수 있다고 판단했습니다.

### 4. DnCNN

노이즈 자체를 예측한 뒤 입력에서 이를 제거하는 residual learning 방식의 DnCNN을 적용했습니다.  
이미지 denoising 분야에서 자주 사용되는 대표적인 구조이기 때문에 비교 모델로 포함했습니다.

## 학습 및 평가 방식

모든 모델은 동일한 학습 흐름으로 비교했습니다.

- loss function: MSELoss
- optimizer: Adam
- learning rate: 0.0005
- epoch: 40
- device: CUDA 기반 학습

평가 지표는 다음 두 가지를 사용했습니다.

- RMSE
- PSNR

평가는 두 방식으로 진행했습니다.

1. train output과 train_cleaned를 비교하는 validation 성능 확인
2. test output과 test input을 기준으로 모델 간 상대 비교 수행

test에는 정답 이미지가 없기 때문에, 모델 비교를 통해 가장 안정적으로 복원 품질이 좋은 모델을 최종 모델로 선정했습니다.

---

## 모델별 성능 비교

실험 결과는 다음과 같았습니다.

| Model | Train RMSE | Train PSNR | Test RMSE | Test PSNR |
|------|-----------:|-----------:|----------:|----------:|
| DAE | 0.1072 | 19.40 | 0.2088 | 13.66 |
| MultiScale_AE | 0.0573 | 24.85 | 0.1778 | 15.04 |
| U-Net | 0.0360 | 28.87 | 0.1648 | 15.70 |
| DnCNN | 0.0306 | 30.30 | 0.1821 | 14.83 |

비교 결과는 다음과 같이 해석할 수 있었습니다.

- DAE는 기본 복원 기준선으로는 의미가 있었지만 성능은 가장 낮았음
- Multi-Scale Autoencoder는 DAE보다 복원 품질이 크게 향상됨
- DnCNN은 학습 성능은 좋았지만 test 기준 복원 품질에서는 U-Net보다 낮았음
- U-Net은 test RMSE가 가장 낮고 PSNR이 가장 높아 최종 복원 품질이 가장 우수했음

---

## 최종 모델 선정

최종 모델은 U-Net으로 선정했습니다.

선정 이유는 다음과 같습니다.

- 비교한 모델 중 가장 낮은 Test RMSE 기록
- 비교한 모델 중 가장 높은 Test PSNR 기록
- skip connection 구조 덕분에 문서 윤곽과 세부 정보 복원에 유리
- 데이터 수가 많지 않은 환경에서도 안정적으로 높은 성능을 보임

최종 성능은 다음과 같습니다.

- Test RMSE: 0.1648
- Test PSNR: 15.70 dB

이 결과를 바탕으로 U-Net을 사용해 test 문서 전체에 대한 복원을 수행했습니다.

## 이 프로젝트를 통해 갖추게 된 능력

### 1. 이미지 복원 문제를 데이터 구조에 맞게 정의하는 능력

단순한 이미지 분류가 아니라 noisy image를 clean image로 복원하는 image-to-image 문제를 직접 다뤄보면서, 문제 유형에 맞는 데이터 구성과 학습 방식을 설계하는 경험을 쌓을 수 있었습니다.

### 2. 문서 이미지에 맞는 전처리 설계 능력

이미지 크기 불일치를 패딩으로 해결하고, bilateral filter와 grayscale 변환을 적용해 문서 복원 문제에 적합한 입력을 구성했습니다.  
이를 통해 이미지 전처리를 단순한 형식 변환이 아니라 문제 맞춤형 설계 과정으로 다루는 역량을 기를 수 있었습니다.

### 3. 여러 딥러닝 복원 모델을 비교하고 선택하는 능력

DAE, Multi-Scale Autoencoder, U-Net, DnCNN을 직접 구현하고 같은 기준으로 비교하면서, 모델 구조 차이가 실제 복원 품질에 어떤 영향을 주는지 분석할 수 있었습니다.

### 4. PyTorch 기반 비전 모델 구현 능력

Dataset, DataLoader, 사용자 정의 transform, 모델 클래스 정의, 학습 루프, 시각화 함수까지 전체 파이프라인을 직접 구성하면서 PyTorch 기반 컴퓨터 비전 프로젝트 구현 역량을 강화할 수 있었습니다.

### 5. 정량 평가와 정성 평가를 함께 수행하는 능력

RMSE와 PSNR 같은 수치 지표뿐 아니라, 실제 복원 결과 이미지를 직접 확인하며 모델의 품질을 비교했습니다.  
이를 통해 모델 성능을 숫자로만 판단하지 않고 시각적 결과까지 함께 해석하는 습관을 갖게 되었습니다.

### 6. 복원 모델을 실무 관점에서 해석하는 능력

문서 복원은 이후 OCR, 문서 분류, 텍스트 추출 등 후속 작업의 성능에 직접적인 영향을 주는 전처리 단계입니다.  
이번 프로젝트를 통해 복원 모델을 독립적인 과제가 아니라, 전체 문서 처리 파이프라인의 성능을 높이는 핵심 단계로 이해하게 되었습니다.

## 이 프로젝트에서 중요하게 본 점

이번 프로젝트에서 가장 중요하게 생각한 점은 단순히 모델을 많이 돌려보는 것이 아니라, 왜 특정 구조가 문서 복원에 더 잘 맞는지를 비교를 통해 설명할 수 있어야 한다는 것이었습니다.

그래서 다음과 같은 원칙으로 프로젝트를 진행했습니다.

- 먼저 데이터 구조를 정확히 파악하기
- 이미지 크기와 채널을 일관되게 정리하기
- 문서 복원에 맞는 전처리를 적용하기
- 여러 모델을 같은 조건에서 비교하기
- 정량 지표와 시각적 결과를 함께 확인하기
- 최종 모델 선정 근거를 명확히 남기기

이러한 흐름은 향후 이미지 복원, OCR 전처리, 비전 기반 자동화 시스템을 다룰 때 중요한 기반이 될 수 있다고 생각합니다.
