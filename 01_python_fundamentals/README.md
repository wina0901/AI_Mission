# Python 기초, 객체지향, 알고리즘 실습 프로젝트

## 1. 프로젝트 소개

Python의 기초 문법부터 객체지향 프로그래밍, 파일 처리, 알고리즘 문제 해결까지 단계적으로 학습하고 구현한 실습 프로젝트입니다.

단순히 문제를 푸는 데서 끝나는 것이 아니라,  
각 문제를 통해 기본 문법을 실제 코드로 구현하는 능력,  
입력값과 예외 상황을 안정적으로 처리하는 습관,  
그리고 여러 객체가 상호작용하는 구조를 설계하는 경험을 쌓는 것을 목표로 진행했습니다.

특히 이 프로젝트는 기초 역량을 다지는 데 의미가 있습니다.  
실무에서는 복잡한 모델이나 프레임워크보다도, 먼저 정확한 로직 작성, 안정적인 예외 처리, 유지보수 가능한 코드 구조가 중요하다고 생각했고, 그 기반을 만들기 위해 이 미션을 수행했습니다.

---

## 2. 이 프로젝트를 진행한 이유

이 프로젝트를 진행한 가장 큰 이유는,  
Python을 “문법적으로 아는 수준”이 아니라 실제로 구현할 수 있는 수준으로 끌어올리기 위해서였습니다.

처음에는 간단한 반복문, 조건문, 문자열 처리 문제부터 시작했지만,  
점차 예외 처리, 입력 검증, 클래스 설계, 파일 입출력, 그리고 여러 객체가 연결되는 작은 시스템 구현까지 확장하면서  
코드를 단순히 작성하는 것을 넘어 구조적으로 사고하는 연습을 하고자 했습니다.

또한 이후 AI 프로젝트나 데이터 처리 프로젝트를 진행할 때도 결국 필요한 것은  
- 데이터를 다루는 기본 로직
- 오류에 흔들리지 않는 처리 방식
- 객체와 기능을 역할별로 나누는 설계 능력
이기 때문에, 이 미션을 하나의 기초 체력 훈련 과정으로 생각하고 수행했습니다.

---

## 3. 주요 구현 내용

### Python 기초
- 3의 배수 출력
- 모음 제거
- 숫자 크기 비교
- 문자열 길이 확인
- 누적 합 계산
- 짝수 합 계산
- 두 수 나누기 및 예외 처리

### 응용 문제
- 문자열 압축
- 숫자 입력 검증 기반 덧셈 프로그램
- 카운트다운 타이머

### 객체지향 프로그래밍
- `TimeTracker`를 통한 시간 측정
- `Contact` 클래스를 통한 연락처 정보 표현
- `VoteSystem`을 통한 후보 등록 및 투표 집계
- `BankAccount`를 통한 입금/출금 로직 구현
- `EmployeeManager`를 통한 클래스 단위 데이터 관리
- `ReservationSystem`을 통한 예약 데이터 관리

### 심화 문제
- CSV 파일을 읽어 남은 시간 안에 수행 가능한 작업 추천
- 소수 판별 및 누적 합 계산
- `Book`, `Member`, `Rental`, `LibraryManagement` 클래스를 활용한 도서관 관리 시스템 구현

## 4. 이 프로젝트를 통해 갖춘 역량

### 1) 기본 문법을 실제 로직으로 구현하는 능력
반복문, 조건문, 함수, 문자열 처리, 리스트/딕셔너리 같은 기본 요소를  
단순 문법 암기가 아니라 문제 해결 로직으로 연결하는 경험을 쌓았습니다.

### 2) 예외 처리와 입력 검증 능력
나눗셈, 숫자 입력, 날짜 형식 검증 등의 문제를 구현하면서  
잘못된 입력이 들어와도 프로그램이 중단되지 않도록 처리하는 습관을 익혔습니다.  
이를 통해 안정적인 프로그램을 작성하는 기본기를 갖출 수 있었습니다.

### 3) 객체지향 설계 능력
시간 관리, 주소록, 투표 시스템, 은행 계좌, 예약 시스템, 도서관 관리 시스템을 구현하면서  
데이터와 기능을 클래스로 묶고, 각 객체의 책임을 분리하는 연습을 했습니다.  
이를 통해 기능 중심이 아니라 구조 중심으로 코드를 설계하는 능력을 키웠습니다.

### 4) 상태 관리와 데이터 흐름 이해
입금/출금에 따른 잔액 변화, 대여/반납에 따른 상태 변화처럼  
프로그램 안에서 데이터가 어떻게 변하고 연결되는지를 직접 구현하면서  
상태 기반 로직을 설계하는 감각을 익혔습니다.

### 5) 파일 입출력과 데이터 처리 능력
CSV 파일을 읽어 필요한 데이터를 가공하고 정렬하여 활용하는 문제를 통해  
실제 데이터 처리의 기초가 되는 파일 입출력 및 데이터 전처리 능력을 쌓았습니다.

### 6) 작은 규모의 시스템을 끝까지 구현하는 경험
도서관 관리 시스템에서는 도서, 회원, 대여라는 여러 객체를 정의하고  
추가, 대여, 반납, 출력 기능까지 연결해보며  
단순 함수 단위가 아니라 하나의 작은 시스템을 설계하고 구현하는 경험을 얻었습니다.

---

## 5. 프로젝트를 통해 배운 점

이 프로젝트를 통해, 좋은 코드는 단순히 “정답이 나오는 코드”가 아니라  
입력을 검증하고, 예외를 고려하고, 기능을 역할별로 나누고,  
나중에 확장하기 쉬운 구조로 작성된 코드라는 점을 배울 수 있었습니다.

또한 Python 기초 문제라고 하더라도,  
그 안에는 실무 개발에 필요한 핵심 요소인  
로직 설계, 상태 관리, 객체지향 사고, 데이터 처리, 안정성 확보가 모두 포함되어 있다는 점을 체감했습니다.

## 6. 앞으로의 확장 방향

이 프로젝트는 Python 기초와 객체지향 설계의 기반을 다지는 과정이었고,  
앞으로는 이를 바탕으로 다음 단계로 확장해보고자 합니다.

- 파일 기반 로직을 데이터베이스 연동 구조로 확장
- 클래스 기반 프로그램을 API 형태로 확장
- 더 복잡한 데이터 처리 및 자동화 로직 구현
- AI/데이터 프로젝트에서 재사용 가능한 구조로 발전
