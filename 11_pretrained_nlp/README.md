# 한영 기계번역 프로젝트

## 프로젝트 개요

이 프로젝트는 한국어 문장을 영어로 번역하는 Neural Machine Translation 프로젝트입니다.  
단순히 번역 모델 하나를 학습시키는 데서 끝나지 않고, 기본 Seq2Seq 구조에서 시작해 Attention, 양방향 인코더, Packed Sequence, Scheduler, Beam Search까지 단계적으로 확장하며 번역 성능이 어떻게 향상되는지를 비교하는 방식으로 진행했습니다.

이번 프로젝트에서는 한국어 입력과 영어 출력으로 구성된 병렬 코퍼스를 활용해 직접 전처리, vocabulary 구축, Dataset / DataLoader 구성, Seq2Seq 모델 구현, 학습 및 평가, BLEU 기반 성능 비교까지 전체 번역 파이프라인을 직접 구성했습니다.  
또한 단순히 최종 점수만 확인하는 것이 아니라, 구조를 하나씩 개선해가며 어떤 요소가 실제 번역 품질 향상에 기여하는지를 실험적으로 검증했습니다.

---

## 이 프로젝트를 왜 했는가

기계번역은 자연어처리에서 가장 대표적인 생성형 문제 중 하나입니다.  
텍스트 분류처럼 입력을 하나의 라벨로 바꾸는 문제가 아니라, 입력 문장의 의미를 유지하면서 다른 언어의 새로운 문장을 생성해야 하기 때문에 더 복잡한 구조와 문맥 이해가 필요합니다.

이번 프로젝트는 다음과 같은 이유로 진행했습니다.

- 자연어처리에서 분류를 넘어 생성형 문제를 직접 다뤄보고 싶었습니다.
- Seq2Seq와 Attention이 왜 등장했는지, 그리고 실제로 성능 차이가 얼마나 나는지 직접 구현해 확인해보고 싶었습니다.
- 단방향 인코더와 양방향 인코더의 차이를 실험적으로 비교해보고 싶었습니다.
- Packed Sequence, Learning Rate Scheduler, Beam Search 같은 학습 및 디코딩 개선 기법이 실제 번역 품질 향상에 어떤 영향을 주는지 경험해보고 싶었습니다.
- 단순히 결과만 내는 것이 아니라, 기본 구조에서 시작해 점진적으로 성능을 개선하는 실험 설계 과정을 포트폴리오로 남기고 싶었습니다.

즉, 이번 프로젝트는 단순한 번역 모델 실습이 아니라, 기계번역 모델의 핵심 구조를 단계적으로 구현하고 개선하면서 생성형 NLP 모델의 동작 원리와 성능 향상 과정을 실전적으로 익히기 위해 진행했습니다.

---

## 문제 정의

- 주제: 한국어 → 영어 기계번역
- 문제 유형: Sequence-to-Sequence Text Generation
- 입력: 한국어 문장
- 출력: 영어 번역 문장
- 주요 평가 지표:
  - Validation Loss
  - sacreBLEU

이번 프로젝트의 핵심은 한국어 문장을 단순히 단어 대응으로 바꾸는 것이 아니라, 문맥을 반영한 자연스러운 영어 문장으로 생성하는 것이었습니다.

---

## 데이터 설명

사용한 데이터는 한국어-영어 일상 대화 병렬 코퍼스입니다.

데이터 특성은 다음과 같습니다.

- train set과 validation set이 JSON 형태로 제공
- 각 샘플은 한국어 문장과 영어 번역 문장으로 구성
- 노트북에서는 학습 효율을 위해 일부 샘플 수를 설정해 사용
  - Train: 60,000개
  - Validation: 3,000개
- 문장 길이 제한을 두어 지나치게 긴 샘플은 제외

이번 프로젝트에서는 병렬 코퍼스를 그대로 사용하는 것이 아니라, 토큰 길이 분석을 먼저 수행한 뒤 적절한 최대 길이를 설정하고 길이 제한을 넘는 문장쌍은 필터링했습니다.

즉, 번역 모델이 학습 가능한 범위 안에서 최대한 많은 문장 패턴을 반영하도록 데이터 전처리를 설계한 프로젝트였습니다.

---

## 프로젝트 진행 과정

### 1. 데이터 로드 및 문장 길이 분석

먼저 train / validation JSON 파일을 불러오고, 한국어 문장과 영어 문장을 각각 분리했습니다.  
이후 한국어와 영어 토큰 길이 분포를 확인해 몇 토큰까지를 최대 길이로 둘지 결정했습니다.

이 과정의 목적은 단순히 데이터를 읽는 것이 아니라, 너무 긴 문장이 학습을 불안정하게 만들지 않도록 데이터 길이 특성을 먼저 이해하는 것이었습니다.

---

### 2. 토큰화 및 길이 기반 필터링

한국어와 영어는 서로 다른 방식으로 토큰화했습니다.

- 한국어: Kiwi 형태소 분석기 사용
- 영어: NLTK word_tokenize 사용

이후 문장 길이가 MAX_LENGTH 기준을 넘는 경우는 제외하고, 길이 조건을 만족하는 문장쌍만 학습에 사용했습니다.

이 과정은 단순 전처리가 아니라, 번역 모델이 지나치게 긴 시퀀스로 인해 학습이 흔들리지 않도록 입력 길이를 통제하는 설계 단계였습니다.

---

### 3. Vocabulary 구축

번역 모델에 문장을 넣기 위해 각 언어별 vocabulary를 직접 구축했습니다.

구성한 주요 요소는 다음과 같습니다.

- SOS 토큰
- EOS 토큰
- PAD 토큰
- UNK 토큰

각 문장을 토큰 시퀀스로 바꾸고, 단어를 인덱스로 매핑한 뒤, 문장 앞뒤에 시작/끝 토큰을 붙여 모델 입력 형태로 변환했습니다.

즉, 이번 프로젝트는 외부 토크나이저 결과를 그대로 쓰는 것이 아니라, 번역 모델용 vocabulary와 시퀀스 인코딩 구조를 직접 설계한 프로젝트였습니다.

---

### 4. Dataset / DataLoader 구성

문장쌍을 모델 학습용 데이터셋으로 구성하기 위해 TranslationDataset 클래스를 직접 구현했습니다.

이 과정에서 다음을 처리했습니다.

- 한국어 문장 인코딩
- 영어 문장 인코딩
- 길이 정보 저장
- 배치 단위 패딩 처리

즉, 단순한 텍스트 리스트가 아니라, 번역 모델이 바로 사용할 수 있는 시퀀스 데이터셋 구조를 end-to-end로 직접 구성했습니다.

---

## 모델링 전략

이번 프로젝트에서는 번역 모델을 한 번에 복잡하게 만들지 않고, 단계적으로 구조를 확장하며 성능을 비교했습니다.

비교한 모델은 다음과 같습니다.

### 1. baseline_seq2seq
가장 기본적인 GRU 기반 Seq2Seq 모델입니다.

- Encoder: 단방향 GRU
- Decoder: 일반 GRU Decoder
- Attention 없음

이 모델은 전체 프로젝트의 기준선 역할을 했습니다.

### 2. attention_seq2seq
기본 Seq2Seq에 Attention을 추가한 모델입니다.

- Encoder: 단방향 GRU
- Decoder: Attention Decoder

이 실험을 통해 인코더의 마지막 hidden state 하나에만 의존하는 구조보다, source 문장의 각 시점 정보를 참조하는 구조가 실제 번역에 얼마나 유리한지 확인하고자 했습니다.

### 3. bigru_attention
인코더를 양방향 GRU로 확장한 모델입니다.

- Encoder: BiGRU
- Decoder: Attention Decoder

이 구조를 통해 입력 문장의 앞뒤 문맥을 모두 반영할 수 있도록 했고, 한국어처럼 어순과 문맥 의존성이 큰 문장에서 성능 향상을 기대했습니다.

### 4. bigru_attention_tuned
기본 BiGRU + Attention 모델에 하이퍼파라미터와 학습 전략을 개선한 버전입니다.

주요 변경점은 다음과 같습니다.

- hidden size 증가
- layer 수 증가
- dropout 증가
- packed sequence 사용
- scheduler 사용
- beam search 적용
- weight decay 적용

### 5. bigru_attention_improved
가장 최종적으로 개선한 실험 버전입니다.

- hidden size: 512
- num_layers: 2
- dropout: 0.3
- learning rate: 0.0005
- packed sequence 사용
- scheduler 사용
- beam width: 5

이 모델은 이전 tuned 버전보다 더 안정적인 학습과 더 나은 번역 품질을 목표로 구성했습니다.

---

## 학습 및 평가 방식

모든 실험은 공통적으로 다음과 같은 방식으로 진행했습니다.

- loss: CrossEntropyLoss
- PAD 토큰은 loss 계산에서 제외
- gradient clipping 적용
- early stopping 적용
- validation loss 기준으로 best checkpoint 선택
- 최종 번역 성능은 sacreBLEU로 평가

또한 단순 숫자만 확인하지 않고, 실제 번역 예시를 함께 출력해 번역 결과가 얼마나 자연스러운지 정성적으로도 비교했습니다.

---

## 모델별 실험 결과

최종 비교 결과는 다음과 같습니다.

| Experiment | Best Valid Loss | sacreBLEU |
|-----------|----------------:|----------:|
| baseline_seq2seq | 3.6108 | 4.2078 |
| attention_seq2seq | 3.1036 | 13.2393 |
| bigru_attention | 2.9959 | 16.2327 |
| bigru_attention_tuned | 2.5238 | 23.6349 |
| bigru_attention_improved | 2.4879 | 24.9528 |

이 결과는 다음과 같이 해석할 수 있습니다.

- 기본 Seq2Seq만으로는 번역 품질이 매우 낮았음
- Attention 추가만으로 BLEU가 크게 상승했음
- BiGRU 인코더를 사용하자 성능이 추가로 향상됨
- packed sequence, scheduler, beam search, hidden size 확장 등 학습 안정화와 디코딩 개선을 반영한 tuned / improved 버전에서 가장 높은 성능을 기록함

즉, 이번 프로젝트는 단순히 최종 모델 하나가 좋았다는 결과보다, 구조와 학습 전략을 하나씩 개선할수록 번역 성능이 실제로 향상된다는 점을 단계적으로 보여준 실험이라는 데 의미가 있습니다.

---

## 최종 모델

최종 선택 모델은 bigru_attention_improved 입니다.

선정 이유는 다음과 같습니다.

- 가장 낮은 validation loss 기록
  - best_valid_loss: 2.4879
- 가장 높은 sacreBLEU 기록
  - sacreBLEU: 24.9528
- attention, 양방향 인코더, packed sequence, scheduler, beam search까지 포함한 가장 완성도 높은 구조
- 이전 tuned 버전보다도 더 안정적인 일반화 성능 확보

즉, 이번 프로젝트에서는 단순히 구조만 복잡하게 만든 것이 아니라, 실제 성능 검증을 통해 가장 효과적인 조합을 최종 모델로 선정했습니다.

---

## 번역 예시 해석

노트북에서 출력된 실제 번역 예시를 보면 다음과 같은 흐름을 확인할 수 있었습니다.

- baseline_seq2seq는 짧은 문장도 자주 어색하거나 엉뚱한 번역을 생성
- attention_seq2seq부터 핵심 문장 구조를 조금 더 반영하기 시작
- bigru_attention은 짧은 문장 번역 품질이 확실히 개선됨
- tuned / improved 모델은 문장 표현이 더 자연스러워지고, 짧은 문장에서는 비교적 안정적인 번역을 생성

다만 긴 문장에서는 여전히 반복 표현이나 의미 왜곡이 남아 있었기 때문에, RNN 기반 Seq2Seq 구조의 한계도 함께 확인할 수 있었습니다.

즉, 이번 프로젝트는 성능 향상을 확인하는 동시에, 전통적 Seq2Seq 구조가 긴 문장에서 어떤 약점을 가지는지도 직접 관찰한 프로젝트였습니다.

## 이 프로젝트를 통해 갖추게 된 능력

### 1. 기계번역 파이프라인을 end-to-end로 구현하는 능력

데이터 로드, 토큰화, vocabulary 구축, 문장 인코딩, Dataset / DataLoader 구성, 모델 학습, 번역 생성, BLEU 평가까지 전체 기계번역 파이프라인을 직접 구현할 수 있게 되었습니다.

### 2. Seq2Seq와 Attention 구조를 실제로 이해하고 구현하는 능력

단순한 개념 학습이 아니라, 기본 Seq2Seq와 Attention Seq2Seq를 직접 비교하면서 왜 Attention이 필요한지 실험적으로 이해할 수 있었습니다.

### 3. 양방향 인코더의 효과를 검증하는 능력

BiGRU 인코더를 도입해 앞뒤 문맥을 동시에 반영하도록 설계했고, 실제로 성능이 향상되는 것을 확인했습니다.  
이를 통해 문맥 표현 강화가 번역 품질 향상에 어떤 영향을 주는지 실전적으로 이해할 수 있었습니다.

### 4. 학습 안정화 및 디코딩 개선 기법 적용 능력

Packed Sequence, Learning Rate Scheduler, Gradient Clipping, Beam Search를 적용하며 단순 모델 구현을 넘어 학습 안정성과 추론 품질을 높이는 방법을 경험할 수 있었습니다.

### 5. 생성형 NLP 모델을 단계적으로 개선하는 실험 설계 능력

이번 프로젝트는 처음부터 복잡한 모델을 쓰지 않고, baseline → attention → BiGRU → tuned → improved 순서로 구조를 확장했습니다.  
이를 통해 모델 개선 과정을 실험적으로 설계하고, 각 요소의 기여도를 분리해 확인하는 역량을 기를 수 있었습니다.

### 6. 번역 품질을 정량·정성적으로 함께 평가하는 능력

Validation loss와 sacreBLEU 같은 정량 지표뿐 아니라, 실제 번역 문장을 직접 비교하며 결과를 해석했습니다.  
이를 통해 생성형 모델 결과를 숫자와 실제 출력 품질 양쪽에서 함께 보는 습관을 갖게 되었습니다.

---

## 이 프로젝트에서 중요하게 본 점

이번 프로젝트에서 가장 중요하게 생각한 점은 최종 성능만 제시하는 것이 아니라, 왜 성능이 올라갔는지를 단계적으로 설명할 수 있어야 한다는 것이었습니다.

그래서 다음과 같은 원칙으로 프로젝트를 진행했습니다.

- 번역 문제에 맞는 토큰화와 vocabulary를 직접 구성하기
- 길이 분포를 보고 적절한 max length를 설정하기
- baseline 모델을 먼저 두고 구조 확장을 비교하기
- attention의 효과를 단독으로 검증하기
- 양방향 인코더와 packed sequence 같은 개선 요소를 단계적으로 반영하기
- BLEU와 실제 번역 예시를 함께 보며 평가하기

이러한 흐름은 향후 기계번역, 요약, 질의응답, 챗봇 같은 생성형 자연어처리 프로젝트를 수행할 때 중요한 기반이 된다고 생각합니다.

## 개선 방향

이번 결과를 바탕으로 향후 다음과 같은 방향으로 성능을 더 높일 수 있다고 판단했습니다.

- SentencePiece 또는 BPE 기반 subword 토크나이징 적용
- Transformer 기반 번역 모델과 비교
- teacher forcing ratio 조정
- label smoothing 적용
- 더 큰 병렬 코퍼스 활용
- BLEU 외에 ROUGE, chrF 등 추가 지표 비교
- detokenization을 반영한 더 정확한 BLEU 측정

즉, 현재 모델은 RNN 기반 Seq2Seq 구조 안에서는 의미 있는 성능 향상을 보여주었지만, 더 강한 문맥 표현과 서브워드 기반 처리 기법을 적용하면 추가 개선 가능성이 충분하다고 볼 수 있습니다.
