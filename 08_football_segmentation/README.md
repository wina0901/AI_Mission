# 축구 경기 장면 Semantic Segmentation 프로젝트

## 프로젝트 개요

이 프로젝트는 축구 경기 이미지에서 선수, 심판, 골대, 관중, 배경 등 다양한 객체를 픽셀 단위로 구분하는 Semantic Segmentation 프로젝트입니다.  
단순히 이미지 전체를 분류하는 것이 아니라, 이미지의 모든 픽셀에 대해 어떤 클래스에 속하는지를 예측하는 문제를 다뤘습니다.

이번 프로젝트에서는 U-Net 계열 모델들을 직접 구현하고 비교하면서, 축구 경기 장면처럼 객체가 많고 경계가 복잡한 이미지에서 어떤 구조가 더 좋은 분할 성능을 보이는지 검증했습니다.  
또한 Optuna를 활용한 하이퍼파라미터 튜닝, CrossEntropy와 Dice Loss를 결합한 손실 설계, 조기 종료, mIoU 및 mDice 기반 평가, 예측 마스크 시각화까지 전체 분할 파이프라인을 직접 구성했습니다.

---

## 이 프로젝트를 왜 했는가

이미지 분류는 이미지 단위로 하나의 정답을 예측하는 문제이지만, 실제 컴퓨터 비전에서는 객체가 어디에 있고, 어떤 영역을 차지하는지까지 이해해야 하는 경우가 많습니다.  
특히 스포츠 영상 분석, 자율주행, 의료영상, 산업 비전처럼 장면 이해가 중요한 문제에서는 픽셀 단위 분할이 매우 핵심적인 기술입니다.

이번 프로젝트는 다음과 같은 이유로 진행했습니다.

- 이미지 분류나 객체 탐지를 넘어, 픽셀 단위로 장면을 이해하는 Semantic Segmentation 문제를 직접 다뤄보고 싶었습니다.
- 축구 경기 장면처럼 다양한 객체가 한 화면에 섞여 있는 복잡한 환경에서 분할 모델이 어떻게 동작하는지 경험해보고 싶었습니다.
- U-Net, U-Net++, Attention U-Net처럼 구조가 다른 분할 모델을 비교하면서, skip connection과 attention 구조의 실제 효과를 확인하고 싶었습니다.
- 단순히 모델을 학습하는 데서 끝나는 것이 아니라, mIoU와 mDice 같은 분할 전용 지표를 직접 계산하고 해석하는 경험을 쌓고 싶었습니다.
- 하이퍼파라미터 튜닝과 시각화를 포함한 전체 segmentation 실험 흐름을 직접 완성해보고 싶었습니다.

즉, 이번 프로젝트는 단순한 이미지 처리 실습이 아니라, 장면 이해 문제를 픽셀 단위로 해결하는 컴퓨터 비전 프로젝트를 직접 설계하고 검증해보기 위해 진행했습니다.

---

## 문제 정의

- 주제: 축구 경기 장면 Semantic Segmentation
- 문제 유형: 멀티클래스 픽셀 단위 분할
- 입력: 축구 경기 이미지
- 출력: 각 픽셀의 클래스 예측 마스크
- 클래스 수: 11개
- 주요 평가 지표:
  - mIoU
  - mDice
  - segmentation loss

이번 프로젝트의 핵심은 이미지 전체를 맞추는 것이 아니라, 각 픽셀의 정답 클래스를 얼마나 정확하게 구분할 수 있는지를 평가하는 것이었습니다.

---

## 데이터 설명

사용한 데이터는 football semantic segmentation 데이터셋이며, 원본 이미지와 segmentation 마스크가 쌍으로 구성되어 있었습니다.

데이터 특성은 다음과 같습니다.

- 전체 이미지 쌍: 100개
- 원본 이미지와 fuse 마스크 이미지가 1:1 대응
- 마스크는 RGB 색상값으로 클래스가 구분되어 있음
- 최종 클래스 수: 11개
- train / val / test 분할:
  - Train: 70장
  - Validation: 15장
  - Test: 15장

이번 프로젝트에서는 마스크 이미지에 존재하는 고유 색상을 직접 수집한 뒤, 색상별 라벨 인덱스를 매핑하여 segmentation 학습용 마스크로 변환했습니다.

---

## 프로젝트 진행 과정

### 1. 데이터 탐색 및 마스크 구조 확인

먼저 원본 이미지와 마스크 이미지가 정상적으로 짝을 이루는지 확인하고, 샘플 이미지를 직접 시각화해 데이터 구조를 점검했습니다.  
이후 마스크 전체를 순회하며 고유한 색상값을 수집했고, 이를 기반으로 색상-클래스 라벨 매핑을 구성했습니다.

이 단계에서 확인한 핵심은 다음과 같습니다.

- 이미지와 마스크가 정상적으로 1:1 대응되는지 확인
- 마스크가 단일 채널이 아니라 RGB 색상 기반으로 클래스 정보를 담고 있다는 점 확인
- 전체 클래스 수가 11개임을 확인

이 과정은 단순 로딩이 아니라, segmentation 문제에 맞게 마스크를 해석 가능한 형태로 바꾸기 위한 준비 단계였습니다.

---

### 2. 데이터 전처리 및 증강

분할 모델이 안정적으로 학습할 수 있도록 이미지와 마스크에 맞는 전처리를 적용했습니다.

적용한 주요 전처리는 다음과 같습니다.

- 이미지 크기를 256x256으로 통일
- train 이미지에는 ColorJitter 적용
- train 단계에서 horizontal flip 적용
- ImageNet 기준 mean / std 정규화 적용
- 마스크는 interpolation 시 클래스 값이 깨지지 않도록 nearest interpolation 사용

특히 segmentation에서는 이미지와 마스크가 항상 동일한 방식으로 변환되어야 하기 때문에, resize와 flip을 이미지와 마스크에 함께 적용하도록 직접 구현했습니다.

---

### 3. 손실 함수와 평가 함수 설계

이번 프로젝트에서는 segmentation 문제에 맞는 학습과 평가를 위해 주요 함수들을 직접 구성했습니다.

구현한 핵심 요소는 다음과 같습니다.

- confusion matrix 계산 함수
- mIoU 계산 함수
- mDice 계산 함수
- DiceLoss
- CrossEntropy + Dice 결합 손실
- EarlyStopping
- train / evaluate 루프

손실 함수는 CrossEntropy만 사용하는 대신 Dice Loss를 함께 결합했습니다.  
그 이유는 segmentation에서는 클래스별 픽셀 겹침 정도를 잘 반영하는 Dice 계열 손실이 성능 향상에 도움을 줄 수 있기 때문입니다.

즉, 이번 프로젝트는 단순히 기본 손실 함수를 쓰는 것이 아니라, segmentation 문제 특성에 맞게 학습 목표를 설계한 프로젝트였습니다.

---

## 모델링 전략

이번 프로젝트에서는 U-Net 계열의 세 가지 모델을 비교했습니다.

### 1. U-Net Deep

기본 U-Net 구조를 더 깊게 확장한 모델입니다.  
인코더-디코더 구조와 skip connection을 통해 저수준 공간 정보와 고수준 의미 정보를 함께 활용하도록 구성했습니다.

### 2. U-Net++

Nested skip connection 구조를 활용해, 인코더와 디코더 사이의 feature gap을 줄이도록 설계된 모델입니다.  
보다 정교한 feature fusion이 가능하다는 점에서 비교 대상으로 포함했습니다.

### 3. Attention U-Net

기본 U-Net 구조에 Attention Gate를 추가한 모델입니다.  
중요한 영역에 더 집중하고 덜 중요한 배경 정보는 줄이는 방식으로 segmentation 품질 향상을 기대할 수 있는 구조입니다.

이번 실험은 단순히 어떤 백본이 더 좋은지 보는 것이 아니라, skip connection 강화와 attention 메커니즘이 실제 segmentation 품질에 어떤 차이를 만드는지 비교하는 데 의미가 있었습니다.

## 학습 결과

최종 실험 결과는 다음과 같습니다.

| Model | Best Val mIoU | Test Loss | Test mIoU | Test mDice |
|------|--------------:|----------:|----------:|-----------:|
| Attention U-Net | 0.7612 | 0.5770 | 0.7451 | 0.8157 |
| U-Net++ | 0.7790 | 0.8394 | 0.7448 | 0.8159 |
| U-Net Deep | 0.7547 | 0.5212 | 0.7402 | 0.8127 |

결과를 해석하면 다음과 같습니다.

- Validation 기준으로는 U-Net++이 가장 높은 mIoU를 기록
- Test 기준 mIoU는 Attention U-Net이 가장 높음
- Test mDice는 U-Net++이 근소하게 가장 높음
- 세 모델 모두 큰 차이 없이 비슷한 수준의 성능을 보였지만, Attention 구조와 nested skip connection이 기본 U-Net보다 약간 더 유리한 경향을 보임

즉, 복잡한 경기 장면 segmentation에서는 단순한 U-Net보다 feature fusion을 강화한 구조가 조금 더 안정적인 성능을 낼 수 있음을 확인했습니다.

## 최종 모델 해석

이번 프로젝트에서는 단일 지표만으로 모델을 판단하지 않고, validation과 test 결과를 함께 해석했습니다.

### Validation 기준 최적 모델
U-Net++

- best val mIoU: 0.7790
- validation 기준으로 가장 안정적인 성능

### Test 기준 최적 모델
Attention U-Net

- test mIoU: 0.7451
- 실제 test segmentation 일반화 성능이 가장 높음

### 전체 비교 해석
- U-Net++은 validation에서 가장 우수했고, test mDice도 가장 높았음
- Attention U-Net은 test mIoU에서 가장 높아 실제 분할 성능이 가장 좋았음
- U-Net Deep도 충분히 경쟁력 있었지만, 복잡한 feature 전달 구조를 가진 두 모델에 비해 약간 낮은 성능을 보였음

이 프로젝트에서는 실제 일반화 성능을 더 중요하게 보고, Attention U-Net을 최종적으로 가장 우수한 test 모델로 해석할 수 있습니다.

## 시각화 및 정성 평가

정량 지표뿐 아니라, test set 전체에 대해 모델별 예측 마스크를 시각화해 비교했습니다.

시각화에서는 다음을 함께 확인했습니다.

- 입력 이미지
- 실제 마스크(GT)
- 예측 마스크
- overlay 결과

이를 통해 숫자로는 비슷해 보이는 모델들도 실제 경계 복원, 작은 객체 분리, 배경 구분에서 어떤 차이가 있는지 직접 비교할 수 있었습니다.

즉, 이번 프로젝트는 점수만 비교한 것이 아니라, segmentation 결과가 시각적으로 얼마나 자연스럽고 정확한지도 함께 평가한 프로젝트였습니다.

## 이 프로젝트를 통해 갖추게 된 능력

### 1. Semantic Segmentation 문제를 end-to-end로 구현하는 능력

이미지-마스크 데이터셋 구성부터 전처리, 학습, 평가, 시각화까지 segmentation 프로젝트 전체 흐름을 직접 구현할 수 있게 되었습니다.

### 2. RGB 마스크를 학습용 클래스 마스크로 변환하는 능력

고유 색상값을 수집해 색상-클래스 매핑을 만들고, 이를 픽셀 단위 라벨 마스크로 변환했습니다.  
이를 통해 segmentation 데이터셋 전처리 방식에 대한 실전 경험을 쌓을 수 있었습니다.

### 3. 분할 전용 성능 지표를 해석하는 능력

Accuracy가 아니라 mIoU와 mDice를 중심으로 성능을 평가하고, confusion matrix 기반으로 분할 성능을 계산했습니다.  
이를 통해 segmentation 문제에서 어떤 지표가 왜 중요한지 이해할 수 있었습니다.

### 4. U-Net 계열 모델 비교 능력

U-Net Deep, U-Net++, Attention U-Net을 직접 구현하고 비교하면서, skip connection 강화와 attention 메커니즘이 실제 segmentation 성능에 어떤 영향을 주는지 분석할 수 있었습니다.

### 5. 손실 함수 설계 능력

CrossEntropy와 Dice Loss를 결합해 segmentation 문제에 적합한 학습 목표를 설계했습니다.  
즉, 단순한 기본 loss 사용이 아니라 문제 특성에 맞는 학습 전략을 구성하는 경험을 쌓을 수 있었습니다.

### 6. 하이퍼파라미터 튜닝 능력

Optuna를 활용해 모델별 learning rate, weight decay, beta 값, dice weight를 탐색하며 체계적으로 성능을 개선하는 과정을 경험했습니다.

### 7. 정량 평가와 정성 평가를 함께 수행하는 능력

mIoU, mDice 같은 수치 지표와 실제 예측 마스크 시각화를 함께 보며, 모델 성능을 숫자와 시각적 결과 양쪽에서 동시에 해석하는 능력을 기를 수 있었습니다.

## 이 프로젝트에서 중요하게 본 점

이번 프로젝트에서 가장 중요하게 생각한 점은 segmentation 문제에서 모델 구조 차이를 단순 점수 비교가 아니라 실제 일반화 성능과 시각적 품질까지 포함해 해석하는 것이었습니다.

그래서 다음과 같은 원칙으로 프로젝트를 진행했습니다.

- 이미지와 마스크의 대응 관계를 먼저 정확히 검증하기
- segmentation에 맞는 resize와 augmentation을 직접 구현하기
- RGB 마스크를 클래스 마스크로 정확히 변환하기
- mIoU와 mDice를 중심으로 평가하기
- U-Net 계열 구조 차이를 같은 조건에서 비교하기
- Optuna로 모델별 최적 하이퍼파라미터를 탐색하기
- 예측 결과를 시각화해 정성 평가까지 함께 수행하기

이러한 흐름은 향후 의료영상 분할, 자율주행 장면 분할, 산업용 비전 검사처럼 픽셀 단위 이해가 중요한 문제를 다룰 때 중요한 기반이 된다고 생각합니다.

## 개선 방향

이번 결과를 바탕으로 향후 다음과 같은 방향으로 성능을 더 높일 수 있다고 판단했습니다.

- 데이터 수가 적기 때문에 더 다양한 augmentation 적용
- 클래스 불균형이 존재한다면 클래스 가중치 반영
- pretrained encoder를 사용하는 segmentation 모델 실험
- DeepLabV3+, SegFormer 같은 다른 구조와 비교
- 더 긴 학습과 scheduler 실험
- 클래스별 IoU를 따로 분석해 취약 클래스 보완

즉, 현재 모델은 제한된 데이터셋에서도 안정적인 성능을 보였지만, 더 큰 데이터와 pretrained backbone을 활용하면 추가 개선 가능성이 충분하다고 볼 수 있습니다.

## 한 줄 정리

이 프로젝트는 축구 경기 장면을 픽셀 단위로 분할하기 위해 RGB 마스크 전처리, U-Net 계열 모델 구현 및 비교, CrossEntropy+Dice 손실 설계, Optuna 하이퍼파라미터 튜닝, mIoU 및 mDice 평가, 시각화까지 전 과정을 수행한 Semantic Segmentation 프로젝트입니다.
