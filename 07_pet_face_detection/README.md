# 개·고양이 얼굴 객체 탐지 프로젝트

## 프로젝트 개요

이 프로젝트는 개와 고양이 이미지에서 얼굴 영역을 자동으로 탐지하는 Object Detection 프로젝트입니다.  
단순히 동물 종류를 분류하는 것이 아니라, 이미지 안에서 얼굴이 어디에 있는지를 Bounding Box로 찾아내는 문제를 다뤘습니다.

이번 프로젝트에서는 SSD300 기반의 객체 탐지 모델을 활용하여 개와 고양이 얼굴을 탐지했고, 데이터 전처리부터 XML 어노테이션 파싱, 커스텀 데이터셋 구성, 학습, mAP 평가, Confusion Matrix 분석, 정밀도·재현율 해석, 추론 시각화까지 전체 탐지 파이프라인을 직접 구현했습니다.

특히 단순히 탐지 결과를 확인하는 수준을 넘어서, mAP50과 mAP75, 그리고 mAP(0.5:0.95)를 함께 비교하면서 탐지는 잘 되었지만 박스 위치 정밀도는 어느 정도 수준인지까지 분석하는 데 집중했습니다.

---

## 이 프로젝트를 왜 했는가

컴퓨터 비전에서 분류 문제는 이미지 전체를 보고 하나의 라벨을 예측하지만, 실제 현업에서는 객체가 어디에 있는지까지 알아야 하는 경우가 훨씬 많습니다.  
예를 들어 얼굴 검출, 불량 부품 검출, 의료영상 병변 탐지, 자율주행 인식 등은 모두 객체의 위치 정보가 중요한 문제입니다.

이번 프로젝트는 이런 이유로 진행했습니다.

- 이미지 분류를 넘어 객체 탐지 문제를 직접 다뤄보고 싶었습니다.
- 이미지와 라벨만 쓰는 것이 아니라, XML 어노테이션을 파싱해 Bounding Box 기반 학습 데이터를 구성하는 경험을 쌓고 싶었습니다.
- SSD와 같은 대표적인 객체 탐지 모델이 실제로 어떤 방식으로 학습되고 평가되는지 이해하고 싶었습니다.
- 단순 시각화가 아니라 mAP, IoU, Precision, Recall, F1-score, Confusion Matrix를 함께 보며 탐지 모델을 해석하는 연습을 하고 싶었습니다.
- 객체를 찾는 것과 객체 위치를 정확히 맞추는 것이 서로 다른 문제라는 점을 직접 실험 결과로 확인해보고 싶었습니다.

즉, 이번 프로젝트는 단순한 동물 이미지 실습이 아니라, 객체 탐지 문제를 실제 평가 기준과 함께 다뤄보며 탐지 성능과 위치 정밀도를 함께 해석하는 경험을 쌓기 위해 진행했습니다.

---

## 문제 정의

- 주제: 개와 고양이 얼굴 영역 탐지
- 문제 유형: Object Detection
- 입력: 개 또는 고양이가 포함된 이미지
- 출력: 얼굴 위치 Bounding Box와 클래스 라벨
  - cat
  - dog
- 사용 모델: SSD300 VGG16
- 주요 평가 지표:
  - mAP50
  - mAP75
  - mAP(0.5:0.95)
  - Precision
  - Recall
  - F1-score
  - Confusion Matrix

이번 프로젝트에서는 단순히 맞췄는지 여부뿐 아니라, 예측 박스가 실제 정답 박스와 얼마나 정확히 겹치는지까지 평가했습니다.

---

## 데이터 설명

사용한 데이터는 Oxford 기반의 Cats and Dogs 이미지 데이터셋이며, 이미지와 함께 XML 형식의 어노테이션 파일이 제공되었습니다.

데이터 구성은 다음과 같은 흐름으로 사용했습니다.

- trainval.txt를 이용해 학습 및 검증용 이미지 목록 구성
- test.txt를 이용해 테스트용 이미지 목록 구성
- 각 이미지에 대응하는 XML 파일을 읽어 Bounding Box 정보 추출
- XML이 없는 이미지는 제외하여 이미지와 어노테이션이 1:1로 대응되도록 정리

이 과정을 통해 단순 이미지 분류 데이터가 아니라, 실제 객체 탐지용 데이터셋 형태로 재구성할 수 있었습니다.

---

## 프로젝트 진행 과정

## 1. 데이터 불러오기 및 구조 확인

먼저 KaggleHub를 통해 데이터를 불러오고, train/validation/test에 사용할 파일 목록을 정리했습니다.  
이후 데이터프레임 형태로 이미지 이름과 종(species) 정보를 확인하며 데이터 구성을 점검했습니다.

추가로 예제 이미지를 직접 시각화해 실제 이미지 상태를 확인했고, XML 파일 존재 여부를 체크하여 누락된 어노테이션이 있는지 먼저 검증했습니다.

이 단계에서 중요한 점은 모델을 바로 학습시키는 것이 아니라, 이미지와 어노테이션이 정상적으로 연결되는지 먼저 확인하는 것이었습니다.

---

## 2. XML 어노테이션 파싱 및 Bounding Box 구성

객체 탐지에서는 라벨만 있는 것이 아니라 객체의 위치 정보가 반드시 필요하므로, XML 파일에서 Bounding Box를 파싱해 학습용 어노테이션 리스트를 구성했습니다.

구성한 정보는 다음과 같습니다.

- 이미지 파일명
- 클래스 라벨
- Bounding Box 좌표
- 이미지 크기 정보

이후 예제 이미지 위에 Bounding Box를 그려보며 어노테이션이 제대로 파싱되었는지도 직접 확인했습니다.

이 과정은 단순한 데이터 로딩이 아니라, 탐지 모델에 맞는 데이터 구조를 직접 만드는 과정이었습니다.

---

## 3. 탐지용 함수 및 평가 로직 구현

이번 프로젝트에서는 모델 학습뿐 아니라, 탐지 성능을 해석하기 위한 여러 함수도 직접 구성했습니다.

구현한 주요 요소는 다음과 같습니다.

- collate_fn_det
- IoU 계산 함수
- mAP 평가 함수
- 예측 시각화 함수
- Detection Confusion Matrix 생성 함수
- Precision / Recall / F1-score 계산 함수

특히 객체 탐지는 일반 분류와 달리 예측 박스와 정답 박스를 IoU 기준으로 매칭해야 하므로, 단순 정확도 계산이 아니라 탐지 문제에 맞는 평가 방식을 직접 구현하고 적용했다는 점이 중요했습니다.

---

## 4. 데이터셋 및 DataLoader 구성

PyTorch 기반 학습을 위해 커스텀 데이터셋 클래스를 직접 구성했습니다.

### VOCDataset
- 이미지와 XML 어노테이션을 함께 불러오는 학습/검증용 데이터셋
- Bounding Box와 클래스 라벨을 모델 입력 형식에 맞게 변환

### TestDataset
- 테스트 이미지 전용 데이터셋
- 추론과 시각화에 활용

또한 train/validation을 8:2 비율로 분리하고, 데이터 증강으로는 horizontal flip을 적용했습니다.  
개와 고양이는 좌우가 뒤집혀도 클래스 의미가 바뀌지 않기 때문에, 객체의 위치 다양성을 늘리는 데 적절한 증강이라고 판단했습니다.

추가로 이미지 크기는 SSD 입력에 맞게 300x300으로 통일했습니다.

---

## 5. 클래스 불균형 대응

훈련 데이터에서 cat과 dog 비율 차이가 존재할 수 있기 때문에, WeightedRandomSampler를 적용해 배치 단위에서 클래스가 보다 균형 있게 반영되도록 구성했습니다.

이 과정을 통해 특정 클래스에 편향된 학습을 줄이고, 두 클래스 모두 안정적으로 탐지할 수 있는 학습 환경을 만들고자 했습니다.

즉, 단순히 데이터를 넣는 것이 아니라, 탐지 문제에서도 클래스 분포를 고려한 학습 전략을 적용해본 프로젝트였습니다.

---

## 6. SSD300 VGG16 모델 적용

모델은 torchvision에서 제공하는 SSD300 VGG16 사전학습 모델을 활용했습니다.

설정 내용은 다음과 같습니다.

- 모델: SSD300 VGG16
- 클래스 수: 3
  - background
  - cat
  - dog
- optimizer: AdamW
- learning rate: 1e-4
- weight decay: 0.0005
- scheduler: StepLR
- score threshold: 0.7

이번 실험에서는 이전에 저장해둔 최적 가중치를 불러온 뒤, 1 epoch 추가 학습을 수행하는 방식으로 진행했습니다.  
이를 통해 학습 시간을 효율적으로 관리하면서도 검증 성능을 다시 확인할 수 있도록 구성했습니다.

---

## 7. 검증 성능 평가

검증 결과는 다음과 같았습니다.

- Train Loss: 0.6425
- Validation mAP(0.5:0.95): 0.7414
- Validation mAP50: 0.9790
- Validation mAP75: 0.8819

이 결과는 다음과 같이 해석할 수 있습니다.

- mAP50이 0.9790으로 매우 높기 때문에, 객체 자체를 찾는 능력은 매우 우수함
- mAP75도 0.8819로 높은 편이지만, mAP50보다 낮아지므로 위치 정밀도는 일부 감소함
- mAP(0.5:0.95)가 0.7414라는 점은 IoU 기준이 엄격해질수록 성능이 떨어진다는 뜻으로, 박스 회귀 정밀도는 추가 개선 여지가 있음을 보여줌

즉, 얼굴을 찾는 것 자체는 잘하지만, 정답 박스와 아주 정밀하게 맞추는 부분에서는 더 개선할 수 있는 여지가 있었습니다.

---

## 8. Confusion Matrix 및 클래스별 성능 분석

Confusion Matrix 결과는 다음과 같았습니다.

| True \ Pred | background | cat | dog |
|------------|-----------:|----:|----:|
| background | 0 | 1 | 15 |
| cat | 11 | 226 | 12 |
| dog | 14 | 0 | 473 |

클래스별 Precision / Recall / F1-score는 다음과 같습니다.

| Class | Precision | Recall | F1-score | Support |
|------|----------:|-------:|---------:|--------:|
| cat | 1.0000 | 0.9496 | 0.9741 | 238 |
| dog | 0.9753 | 1.0000 | 0.9875 | 473 |

전체 Accuracy는 98.31%였습니다.

이 결과를 해석하면 다음과 같습니다.

- cat은 Precision이 1.0000으로, cat이라고 예측한 경우는 거의 모두 정확했음
- cat의 Recall은 0.9496으로 일부 cat 얼굴을 놓친 경우가 존재했음
- dog는 Recall이 1.0000으로, 실제 dog 얼굴을 모두 탐지했음
- dog의 Precision도 0.9753으로 매우 높아 전반적으로 안정적인 탐지 성능을 보였음

즉, 두 클래스 모두 매우 우수한 탐지 성능을 보였고, 특히 dog 클래스는 재현율 측면에서 매우 강한 성능을 보였습니다.

---

## 최종 결과 해석

이번 프로젝트의 핵심 결과는 다음과 같습니다.

- mAP50: 0.9790
- mAP75: 0.8819
- mAP(0.5:0.95): 0.7414
- Accuracy: 98.31%
- cat F1-score: 0.9741
- dog F1-score: 0.9875

이 결과를 통해 다음 두 가지를 확인할 수 있었습니다.

### 1. 객체 탐지 자체는 매우 잘 수행됨
낮은 IoU 기준인 mAP50이 매우 높고, Precision/Recall/F1-score도 모두 높기 때문에 개와 고양이 얼굴을 찾는 능력은 충분히 우수하다고 볼 수 있습니다.

### 2. Bounding Box 위치 정밀도는 추가 개선 가능
IoU 기준이 높아질수록 성능이 감소하는 점을 보면, 객체를 찾는 것과 박스를 매우 정밀하게 맞추는 것은 별개의 문제임을 확인할 수 있습니다.  
즉, 이번 실험은 탐지 성공률은 높았지만 박스 회귀 정밀도 측면에서는 개선 여지가 남아 있었습니다.

## 이 프로젝트를 통해 갖추게 된 능력

### 1. 객체 탐지 문제를 end-to-end로 구현하는 능력

이미지 분류가 아니라 객체 위치까지 예측하는 탐지 문제를 직접 다뤘고, 데이터 준비부터 학습, 평가, 시각화까지 전체 파이프라인을 직접 구성할 수 있었습니다.

### 2. XML 기반 어노테이션 데이터를 다루는 능력

VOC 스타일 XML 어노테이션을 직접 파싱하고, 이미지와 Bounding Box를 1:1로 매칭해 탐지용 데이터셋으로 재구성했습니다.  
이를 통해 실제 객체 탐지 데이터셋 전처리 경험을 쌓을 수 있었습니다.

### 3. IoU와 mAP 기반 평가를 해석하는 능력

객체 탐지는 단순 정확도만으로 평가할 수 없기 때문에, IoU와 mAP50, mAP75, mAP(0.5:0.95)를 함께 보며 성능을 해석했습니다.  
이를 통해 객체를 찾는 성능과 위치를 정확히 맞추는 성능을 분리해서 바라보는 역량을 기를 수 있었습니다.

### 4. PyTorch 탐지 모델 활용 능력

torchvision의 SSD300 VGG16 모델을 활용하고, 커스텀 Dataset, DataLoader, collate function, optimizer, scheduler까지 직접 구성하면서 PyTorch 기반 객체 탐지 프로젝트 구현 역량을 강화할 수 있었습니다.

### 5. 클래스별 성능 분석 능력

Confusion Matrix와 Precision, Recall, F1-score를 통해 cat과 dog 클래스별 성능 차이를 분석했습니다.  
이를 통해 모델 전체 점수만 보는 것이 아니라, 각 클래스에서 어떤 오류가 발생하는지 구체적으로 해석하는 능력을 기를 수 있었습니다.

### 6. 탐지 결과를 정량·정성적으로 함께 해석하는 능력

수치 지표뿐 아니라 예측 Bounding Box를 실제 이미지에 시각화하며 결과를 확인했습니다.  
즉, 모델 성능을 숫자와 시각적 결과 양쪽에서 함께 검증하는 습관을 갖게 되었습니다.

## 이 프로젝트에서 중요하게 본 점

이번 프로젝트에서 가장 중요하게 생각한 점은 객체를 찾는 것과 객체 위치를 정확히 맞추는 것이 서로 다른 문제라는 점을 직접 검증하는 것이었습니다.

그래서 다음과 같은 원칙으로 프로젝트를 진행했습니다.

- 이미지와 XML 어노테이션 매칭을 먼저 정확히 검증하기
- 탐지 문제에 맞는 데이터셋 구조를 직접 만들기
- 단순 시각화에 그치지 않고 IoU와 mAP로 정량 평가하기
- 클래스별 Precision, Recall, F1-score를 따로 보기
- Confusion Matrix로 실제 오탐과 미탐을 구분하기
- 결과 이미지를 직접 확인해 정성 평가까지 함께 수행하기

이러한 흐름은 향후 얼굴 검출, 결함 검출, 의료영상 병변 탐지, 산업용 비전 검사처럼 위치 정보가 중요한 문제를 다룰 때 중요한 기반이 된다고 생각합니다.

## 개선 방향

이번 결과를 바탕으로 향후 다음과 같은 방향으로 성능을 더 개선할 수 있다고 판단했습니다.

- Anchor box 튜닝
- Bounding Box 회귀 정밀도 개선
- 데이터 증강 다양화
- 더 긴 epoch 학습 및 하이퍼파라미터 조정
- NMS 및 score threshold 최적화

즉, 현재 모델은 객체 탐지 성능은 매우 우수하지만, 박스 위치를 더 정밀하게 맞추는 방향으로 추가 개선 가능성이 있습니다.
