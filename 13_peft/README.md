# KcELECTRA 기반 쇼핑몰 리뷰 감성 분석 프로젝트

## 프로젝트 개요

이 프로젝트는 쇼핑몰 패션 리뷰 데이터를 대상으로 리뷰 문장의 감성을 부정, 중립, 긍정으로 분류하는 한국어 감성 분석 프로젝트입니다.

단순히 사전학습 언어모델을 fine-tuning하는 데서 끝나지 않고, Full Fine-Tuning, Partial Fine-Tuning, LoRA, AdaLoRA, EIA, Prompt Tuning을 같은 데이터와 같은 평가 기준에서 비교했습니다.

이를 통해 단순 성능뿐 아니라 학습 시간, 모델 크기, 학습 파라미터 수, 배포 효율성까지 함께 고려하여 실제 서비스 환경에서 어떤 fine-tuning 전략이 더 적합한지 분석했습니다.

---

## 이 프로젝트를 왜 했는가

실제 서비스에서 감성 분석 모델을 사용할 때는 단순히 정확도만 높은 모델이 항상 좋은 선택은 아닙니다.  
리뷰 데이터처럼 대량으로 쌓이는 텍스트를 분석하려면 모델 성능뿐 아니라 학습 비용, 저장 공간, 배포 효율, 재학습 가능성까지 함께 고려해야 합니다.

이번 프로젝트는 다음과 같은 이유로 진행했습니다.

- 한국어 쇼핑몰 리뷰 데이터를 기반으로 실무형 감성 분석 문제를 경험하고 싶었습니다.
- KcELECTRA 같은 한국어 사전학습 모델을 실제 도메인 데이터에 fine-tuning해보고 싶었습니다.
- Full Fine-Tuning과 PEFT 방식의 성능 차이를 직접 비교해보고 싶었습니다.
- LoRA, AdaLoRA, EIA, Prompt Tuning처럼 경량 fine-tuning 방식이 실제로 얼마나 효과적인지 검증하고 싶었습니다.
- 단순 정확도 비교가 아니라 모델 크기와 학습 시간까지 함께 고려하는 실무형 모델 선택 기준을 세워보고 싶었습니다.

즉, 이번 프로젝트는 단순한 감성 분류 실습이 아니라, 한국어 리뷰 감성 분석 문제에서 성능과 효율을 함께 고려한 fine-tuning 전략 비교 프로젝트입니다.

---

## 문제 정의

- 주제: 쇼핑몰 패션 리뷰 감성 분석
- 문제 유형: 한국어 텍스트 멀티클래스 분류
- 입력: 쇼핑몰 리뷰 텍스트
- 출력:
  - 부정
  - 중립
  - 긍정
- 사용 모델: beomi/KcELECTRA-base
- 주요 평가 지표:
  - Accuracy
  - Macro F1-score
  - Precision
  - Recall
- 추가 비교 요소:
  - 학습 시간
  - 모델 저장 크기
  - 학습 파라미터 비율

---

## 데이터 설명

사용한 데이터는 JSON 형식의 쇼핑몰 패션 리뷰 데이터입니다.

주요 컬럼은 다음과 같습니다.

- RawText: 리뷰 원문
- GeneralPolarity: 감성 레이블
  - -1: 부정
  - 0: 중립
  - 1: 긍정

전체 데이터는 44,398개이며, 감성 분포는 다음과 같았습니다.

| 클래스 | 비율 |
|------|----:|
| 부정 | 20.9% |
| 중립 | 19.6% |
| 긍정 | 59.6% |

긍정 리뷰가 가장 많고, 부정과 중립 리뷰는 상대적으로 적은 불균형 데이터 구조였습니다.  
따라서 단순 Accuracy만으로 평가하면 긍정 클래스에 치우친 모델이 좋아 보일 수 있기 때문에, Macro F1-score와 클래스별 성능도 함께 확인했습니다.

## 프로젝트 진행 과정

### 1. 데이터 로드 및 전처리

먼저 JSON 파일들을 불러오고, 리뷰 텍스트와 감성 레이블을 추출해 DataFrame으로 구성했습니다.

이후 학습에 사용할 수 있도록 다음 작업을 수행했습니다.

- 리뷰 텍스트 추출
- 감성 레이블 변환
- 결측 및 이상 데이터 확인
- 클래스 분포 확인
- 텍스트 길이 분포 확인
- train / test 분할
- stratify 적용으로 클래스 비율 유지

이 과정은 단순히 데이터를 불러오는 것이 아니라, 감성 분석 모델이 학습할 수 있는 구조로 리뷰 데이터를 정리하고 클래스 불균형 문제를 먼저 파악하는 단계였습니다.

---

### 2. 토크나이징

모델은 beomi/KcELECTRA-base를 사용했기 때문에, 해당 모델의 tokenizer를 불러와 리뷰 문장을 토큰화했습니다.

토크나이징 과정에서는 다음을 처리했습니다.

- 리뷰 텍스트를 input_ids로 변환
- attention_mask 생성
- 최대 길이 기준 truncation 적용
- padding 적용
- label 컬럼을 labels로 변환
- Hugging Face Dataset 형식으로 변환

이를 통해 Hugging Face Trainer에서 바로 학습할 수 있는 입력 형태를 구성했습니다.

---

### 3. 클래스 불균형 처리

데이터에서 긍정 리뷰 비율이 높았기 때문에, 모델이 긍정 클래스에 치우쳐 학습될 가능성이 있었습니다.

이를 보완하기 위해 Weighted Cross Entropy Loss를 적용했습니다.

클래스 가중치는 다음과 같습니다.

| 클래스 | 가중치 |
|------|------:|
| 부정 | 1.598 |
| 중립 | 1.704 |
| 긍정 | 0.559 |

중립 클래스가 가장 적고 분류 난이도도 높았기 때문에 가장 높은 가중치를 부여했습니다.  
이를 통해 단순히 다수 클래스인 긍정만 잘 맞추는 모델이 아니라, 부정과 중립 클래스도 함께 고려하는 학습 구조를 만들었습니다.

---

## 비교한 학습 방법

이번 프로젝트에서는 총 6가지 fine-tuning 전략을 비교했습니다.

### 1. Full Fine-Tuning

모델 전체 파라미터를 업데이트하는 방식입니다.  
가장 높은 성능을 기대할 수 있지만, 학습 비용과 저장 공간이 크다는 단점이 있습니다.

### 2. Partial Fine-Tuning

대부분의 레이어를 동결하고 마지막 Transformer 레이어와 분류기만 학습하는 방식입니다.  
학습 속도는 빠르지만, 모델 전체를 저장해야 하므로 저장 공간 측면의 이점은 제한적입니다.

### 3. LoRA

기존 모델 가중치는 고정하고, 저차원 행렬을 추가해 변화량만 학습하는 PEFT 방식입니다.  
적은 파라미터만 학습하면서도 Full Fine-Tuning에 가까운 성능을 낼 수 있는지 확인했습니다.

### 4. AdaLoRA

LoRA의 랭크를 고정하지 않고 학습 중 적응적으로 조절하는 방식입니다.  
더 적은 파라미터로 효율적인 학습이 가능한지 확인했습니다.

### 5. EIA

IA3를 확장한 방식으로, Query, Key, Value, FFN 중간층 등 더 넓은 레이어에 스케일링 벡터를 적용했습니다.  
극히 적은 파라미터로도 도메인 적응이 가능한지 확인했습니다.

### 6. Prompt Tuning

입력 앞에 학습 가능한 soft prompt를 추가하고, 모델 본체는 거의 고정하는 방식입니다.  
가장 적은 파라미터로 학습할 수 있지만, KcELECTRA-base 규모에서 실제 성능이 충분한지 검증했습니다.

## 결과 해석

### Full Fine-Tuning

Full Fine-Tuning은 Accuracy 87.33%, F1-score 84.15%로 가장 높은 성능을 기록했습니다.  
전체 파라미터를 업데이트하기 때문에 데이터에 가장 강하게 적응할 수 있었지만, 저장 크기가 418.7 MB로 크고 학습 비용도 높았습니다.

### LoRA

LoRA는 Accuracy 87.26%, F1-score 83.94%를 기록했습니다.  
Full Fine-Tuning과 Accuracy 차이는 0.07%p에 불과했지만, 모델 크기는 7.0 MB로 크게 줄었습니다.

즉, 거의 동일한 성능을 훨씬 작은 저장 크기로 달성했다는 점에서 성능과 효율의 균형이 가장 뛰어난 방법이었습니다.

### EIA

EIA는 Accuracy 87.16%, F1-score 83.80%를 기록했습니다.  
학습 파라미터는 약 0.05% 수준으로 매우 적었지만 Full Fine-Tuning에 가까운 성능을 보였습니다.

모델 크기도 4.9 MB로 작아, 극한 경량화가 필요한 상황에서 의미 있는 선택지가 될 수 있음을 확인했습니다.

### AdaLoRA

AdaLoRA는 Accuracy 86.66%, F1-score 83.19%로 LoRA보다 약간 낮은 성능을 보였습니다.  
랭크를 적응적으로 조절하는 장점이 있지만, total_step 등 추가 하이퍼파라미터 설정이 필요해 실험 복잡도는 더 높았습니다.

### Partial Fine-Tuning

Partial Fine-Tuning은 에폭당 학습 시간이 6.9분으로 가장 빨랐습니다.  
하지만 Accuracy는 84.77%로 Full Fine-Tuning보다 낮았고, 저장 크기도 전체 모델을 저장하기 때문에 418.7 MB로 동일했습니다.

따라서 빠른 실험에는 유리하지만, 성능과 저장 효율 측면에서는 LoRA나 EIA보다 장점이 적었습니다.

### Prompt Tuning

Prompt Tuning은 Accuracy 59.32%, F1-score 25.17%로 가장 낮은 성능을 보였습니다.  
분류 결과를 보면 부정과 중립 클래스는 거의 맞추지 못하고, 대부분 긍정으로 예측하는 편향이 발생했습니다.

이를 통해 KcELECTRA-base와 같은 110M 규모의 모델에서는 매우 적은 soft prompt 파라미터만으로 감성 분류 태스크에 충분히 적응하기 어렵다는 점을 확인했습니다.

## 클래스별 성능 분석

모든 방법에서 중립 클래스의 F1-score가 가장 낮게 나타났습니다.

| 방법 | Negative F1 | Neutral F1 | Positive F1 |
|------|------------:|-----------:|------------:|
| Full Fine-Tuning | 0.88 | 0.71 | 0.93 |
| Partial Fine-Tuning | 0.85 | 0.67 | 0.92 |
| LoRA | 0.87 | 0.71 | 0.93 |
| AdaLoRA | 0.87 | 0.70 | 0.93 |
| EIA | 0.87 | 0.71 | 0.93 |
| Prompt Tuning | 0.00 | 0.01 | 0.74 |

중립 리뷰는 긍정과 부정 표현이 섞여 있거나, 단순 사실을 서술하는 경우가 많아 경계가 모호했습니다.  
Weighted Loss로 중립 클래스에 가장 높은 가중치를 부여했음에도, 중립 클래스 성능 개선에는 한계가 있었습니다.

이 결과를 통해 감성 분석에서는 단순히 긍정과 부정을 구분하는 것보다, 애매한 중립 표현을 분리하는 것이 더 어려운 문제라는 점을 확인했습니다.

## 최종 결론

| 상황 | 권장 방법 | 이유 |
|------|----------|------|
| 최고 성능이 필요한 경우 | Full Fine-Tuning | 가장 높은 Accuracy와 F1-score |
| 성능과 효율의 균형이 필요한 경우 | LoRA | Full Fine-Tuning과 거의 같은 성능, 60배 수준의 경량화 |
| 극한 경량화가 필요한 경우 | EIA | 매우 적은 파라미터로 높은 성능 달성 |
| 빠른 실험이 필요한 경우 | Partial Fine-Tuning | 에폭당 학습 시간이 가장 짧음 |
| 피해야 할 상황 | Prompt Tuning | 소형 모델 환경에서 성능이 크게 낮음 |

이번 실험에서 가장 현실적인 선택은 LoRA였습니다.  
Full Fine-Tuning과 거의 동일한 성능을 보이면서도 모델 크기를 크게 줄일 수 있었기 때문입니다.

EIA 역시 매우 적은 파라미터로 높은 성능을 보여, 저장 공간이나 배포 비용이 중요한 환경에서 의미 있는 대안이 될 수 있었습니다.

## 이 프로젝트를 통해 갖추게 된 능력

### 1. 한국어 감성 분석 파이프라인 구현 능력

쇼핑몰 리뷰 데이터를 불러오고, 감성 레이블을 정리하고, 토크나이징, 학습, 평가까지 전체 한국어 텍스트 분류 파이프라인을 직접 구현했습니다.

### 2. Hugging Face 기반 사전학습 모델 fine-tuning 능력

KcELECTRA-base 모델을 활용해 한국어 리뷰 도메인에 맞게 fine-tuning했습니다.  
이를 통해 Transformers 기반 모델을 실제 분류 태스크에 적용하는 역량을 기를 수 있었습니다.

### 3. 클래스 불균형 데이터 처리 능력

긍정 리뷰가 많은 불균형 데이터 구조를 확인하고, Weighted Cross Entropy Loss를 적용했습니다.  
이를 통해 다수 클래스에 치우친 학습을 줄이고, 소수 클래스까지 고려하는 학습 전략을 경험했습니다.

### 4. PEFT 기법 비교 및 활용 능력

LoRA, AdaLoRA, EIA, Prompt Tuning을 직접 적용하고 비교했습니다.  
이를 통해 대형 언어모델을 전체 fine-tuning하지 않고도 적은 파라미터로 도메인에 적응시키는 방법을 실험적으로 이해할 수 있었습니다.

### 5. 성능과 효율을 함께 고려한 모델 선택 능력

Accuracy와 F1-score뿐 아니라 학습 시간, 모델 크기, 학습 파라미터 비율까지 함께 비교했습니다.  
이를 통해 실제 서비스 배포 관점에서 어떤 방법이 더 현실적인지 판단하는 능력을 기를 수 있었습니다.

### 6. 클래스별 오류 분석 능력

전체 성능만 확인하지 않고, Negative, Neutral, Positive 클래스별 F1-score를 비교했습니다.  
이를 통해 중립 리뷰가 가장 어려운 클래스라는 점을 확인하고, 감성 분석에서 클래스 경계가 모호한 데이터의 한계를 해석할 수 있었습니다.

### 7. 실험 결과를 전략적으로 정리하는 능력

각 fine-tuning 방식의 장단점을 단순 나열하지 않고, 최고 성능, 경량화, 빠른 실험, 배포 효율이라는 상황별 기준으로 정리했습니다.  
이를 통해 모델 실험 결과를 실무 의사결정에 연결하는 역량을 강화할 수 있었습니다.

## 이 프로젝트에서 중요하게 본 점

이번 프로젝트에서 가장 중요하게 생각한 점은 단순히 가장 높은 정확도를 얻는 것이 아니라, 실제 환경에서 어떤 fine-tuning 전략이 더 적합한지 판단하는 것이었습니다.

그래서 다음과 같은 원칙으로 프로젝트를 진행했습니다.

- 클래스 분포를 먼저 확인하고 불균형을 반영하기
- 동일한 데이터와 평가 지표로 여러 fine-tuning 전략 비교하기
- Accuracy뿐 아니라 Macro F1-score 함께 확인하기
- 모델 크기와 학습 시간까지 함께 비교하기
- 클래스별 F1-score를 분석해 모델의 취약점을 확인하기
- 최종 모델 선택을 성능뿐 아니라 배포 효율 관점에서 해석하기

이러한 흐름은 향후 리뷰 분석, 고객 VOC 분류, 감성 모니터링, 상담 텍스트 분석 같은 실무형 NLP 프로젝트를 수행할 때 중요한 기반이 될 수 있다고 생각합니다.

## 개선 방향

이번 결과를 바탕으로 향후 다음과 같은 방향으로 성능을 더 높일 수 있다고 판단했습니다.

- 중립 클래스 데이터 추가 수집
- 중립 리뷰에 대한 별도 규칙 또는 보조 라벨 설계
- 클래스별 threshold 조정
- focal loss 적용
- 데이터 증강 또는 back translation 적용
- KoELECTRA, KLUE-RoBERTa, KcBERT 등 다른 한국어 모델과 비교
- LoRA rank, alpha, target module 추가 튜닝
- 실제 쇼핑몰 도메인별 리뷰에 대한 성능 비교

특히 중립 클래스 성능 개선이 가장 중요한 과제로 남았습니다.  
중립 리뷰는 표현 자체가 모호한 경우가 많기 때문에, 단순 모델 변경뿐 아니라 데이터 품질과 라벨 기준 개선도 함께 필요하다고 판단했습니다.
