# 고객 정기예금 가입 예측 프로젝트

## 프로젝트 개요

이 프로젝트는 고객의 인구통계 정보, 과거 마케팅 접촉 이력, 그리고 당시의 경제 지표를 바탕으로 고객이 정기예금 상품에 가입할지 여부를 예측하는 이진 분류 프로젝트입니다.

단순히 분류 모델을 하나 적용하는 데서 끝나지 않고, 데이터 탐색, 전처리, 불균형 데이터 처리, 여러 모델 비교, 하이퍼파라미터 튜닝, 최종 모델 선정, 그리고 결과 해석까지 전체 머신러닝 분류 프로젝트의 흐름을 직접 수행하는 데 집중했습니다.

또한 최종적으로는 예측 성능만 확인하는 것이 아니라, 어떤 변수들이 가입 여부에 큰 영향을 주는지 분석하고, 그 결과를 바탕으로 비즈니스적으로 해석 가능한 인사이트까지 도출했습니다.

---

## 이 프로젝트를 왜 했는가

금융 상품 가입 여부 예측은 실제 현업에서 매우 중요한 문제 중 하나입니다.  
특히 마케팅 캠페인에서는 모든 고객에게 같은 방식으로 접근하는 것보다, 가입 가능성이 높은 고객을 선별하고 효율적으로 타겟팅하는 것이 훨씬 중요합니다.

이번 프로젝트는 이러한 실무형 문제를 직접 다뤄보기 위해 진행했습니다.

- 고객 데이터 기반으로 실제 비즈니스 의사결정에 연결될 수 있는 분류 문제를 경험하고 싶었습니다.
- 단순히 정확도를 높이는 것을 넘어, 어떤 특징이 가입 여부를 설명하는지 해석하는 능력을 키우고 싶었습니다.
- 불균형 데이터, 범주형 변수, 경제 지표처럼 실제 데이터에서 자주 마주치는 요소들을 직접 처리해보고 싶었습니다.
- 여러 모델을 비교하고, 성능과 해석 가능성을 함께 고려하여 최종 모델을 선택하는 경험을 쌓고 싶었습니다.

즉, 이번 프로젝트는 단순한 분류 실습이 아니라, 실무형 금융 마케팅 데이터를 바탕으로 예측과 해석을 함께 수행하는 프로젝트를 경험하기 위해 진행했습니다.

---

## 문제 정의

- 주제: 고객의 정기예금 가입 여부 예측
- 문제 유형: 이진 분류
- 목표 변수: y
  - 0 = 미가입
  - 1 = 가입
- 주요 평가 지표: ROC-AUC
- 보조 지표: Accuracy, Precision, Recall, F1-score, Confusion Matrix

이 프로젝트에서 중요한 점은 단순한 정확도보다도, 가입 고객과 미가입 고객을 얼마나 잘 구분해내는가였습니다.  
따라서 클래스 불균형이 있는 상황을 고려해 ROC-AUC를 중심 지표로 설정했습니다.

---

## 데이터 설명

사용한 데이터는 고객의 기본 정보, 기존 마케팅 접촉 정보, 그리고 경제 관련 지표를 함께 포함하고 있었습니다.

주요 변수 예시는 다음과 같습니다.

- 고객 정보: age, job, marital, education, housing, loan
- 마케팅 접촉 정보: campaign, pdays, previous, poutcome, contact, month, day_of_week
- 경제 지표: emp.var.rate, cons.price.idx, cons.conf.idx, euribor3m, nr.employed

이러한 변수들을 바탕으로 고객의 정기예금 가입 여부를 예측했습니다.

## 프로젝트 진행 과정

### 1. EDA를 통한 데이터 이해

먼저 데이터의 구조와 분포를 확인하고, 결측값, unknown 값, 중복값, 수치형/범주형 변수의 특성을 점검했습니다.

주요 확인 내용은 다음과 같습니다.

- 결측값은 없었지만 일부 범주형 변수에는 unknown 값이 존재
- 중복 데이터는 실제 중복으로 판단하여 제거
- 수치형 변수의 분포를 확인하고 로그 변환 또는 구간화가 필요한 변수 탐색
- 범주형 변수의 고유값 분포 확인
- 목표 변수 분포 확인을 통해 클래스 불균형 문제 인지

이 과정을 통해 단순히 데이터를 바로 모델에 넣는 것이 아니라, 각 변수의 의미와 특성을 먼저 파악한 뒤 전처리를 설계하는 흐름을 경험할 수 있었습니다.

---

### 2. 데이터 특성에 맞는 전처리 수행

EDA 결과를 바탕으로 다음과 같은 전처리를 적용했습니다.

수치형 변수 처리
- age는 한쪽으로 치우친 분포를 보여 log1p 변환 수행
- campaign은 이산적 특성을 고려해 구간화 변수 생성
- pdays는 999가 “이전에 연락하지 않음”을 의미하므로 별도 접촉 여부 변수로 분리
- previous, pdays 관련 변수도 의미를 살릴 수 있도록 가공
- duration은 통화가 끝난 뒤에만 알 수 있는 정보이므로 예측 시점과 맞지 않아 제거

범주형 변수 처리
- 범주형 변수는 원-핫 인코딩을 통해 모델 입력 형태로 변환

목표 변수 처리
- y를 0과 1의 이진 값으로 변환하여 분류 문제 형태로 정리

이 과정은 단순 전처리가 아니라, 실제 예측 가능한 정보와 예측 시점의 정보 누수를 구분하고, 변수 의미를 해석한 뒤 모델에 맞는 형태로 변환하는 작업이었습니다.

---

### 3. 불균형 데이터 처리

정기예금 가입 고객은 전체 데이터에서 상대적으로 적은 비율을 차지하고 있었기 때문에, 모델이 다수 클래스에 치우쳐 학습될 가능성이 있었습니다.

이를 해결하기 위해 다음과 같은 방법을 적용했습니다.

- train / valid / test 데이터 분리
- 학습 데이터에만 SMOTETomek 적용
- 클래스 비율을 균형에 가깝게 보정
- 최종 RandomForest 모델에서는 class_weight="balanced"까지 함께 사용

이 과정을 통해 단순 정확도는 높지만 실제 가입 고객을 잘 찾지 못하는 문제를 방지하고자 했습니다.

---

### 4. 다양한 모델 비교

한 가지 모델만 사용하는 대신, 선형 모델부터 트리 기반 앙상블 모델까지 폭넓게 비교했습니다.

비교한 모델은 다음과 같습니다.

- Linear Regression
- Ridge
- Lasso
- Polynomial Regression
- Logistic Regression
- Decision Tree
- Random Forest
- XGBoost
- CatBoost
- LightGBM

특히 분류 문제에 적합한 모델들에 대해 ROC-AUC 기준으로 성능을 비교했고, 그 결과 트리 기반 앙상블 모델들이 전반적으로 더 우수한 성능을 보였습니다.

주요 비교 결과는 다음과 같습니다.

- Logistic Regression ROC-AUC: 0.7331
- Decision Tree ROC-AUC: 0.7676 수준
- XGBoost ROC-AUC: 0.7870
- CatBoost ROC-AUC: 0.7910
- LGBM ROC-AUC: 0.7897
- RandomForest ROC-AUC: 0.7944

이 결과를 통해 고객의 가입 여부는 단순 선형 관계보다, 비선형적이고 복합적인 패턴을 반영하는 트리 계열 모델이 더 잘 설명할 수 있다는 점을 확인했습니다.

---

### 5. Optuna를 활용한 하이퍼파라미터 튜닝

모델 성능을 더 안정적으로 비교하기 위해 Optuna 기반 하이퍼파라미터 튜닝을 진행했습니다.

튜닝 대상은 다음과 같았습니다.

- Decision Tree
- Random Forest
- XGBoost
- CatBoost
- LightGBM

그 결과 RandomForest가 가장 높은 ROC-AUC를 기록했고, 최종 모델로 선정되었습니다.

최적 RandomForest 성능
- Best ROC-AUC: 0.7944

최적 파라미터
- n_estimators: 591
- max_depth: 10
- min_samples_leaf: 14
- min_samples_split: 6
- max_features: log2

이 과정을 통해 감각적인 모델 선택이 아니라, 검증 데이터를 기준으로 객관적으로 최적 모델을 선정하는 흐름을 만들 수 있었습니다.

---

## 최종 모델

최종 모델은 RandomForestClassifier입니다.

선정 이유는 다음과 같습니다.

- 튜닝된 모델들 중 가장 높은 ROC-AUC를 기록
- 불균형 데이터 상황에서도 비교적 안정적인 성능을 보임
- 변수 중요도를 통해 해석 가능성이 높음
- 비선형 관계와 변수 간 상호작용을 잘 반영할 수 있음

최종적으로 train과 validation 데이터를 합친 뒤 다시 학습하고, test 데이터에 대해 평가를 수행했습니다.

최종 Test 성능
- ROC-AUC: 0.7921

분류 리포트 요약
- Accuracy: 0.86
- 가입 클래스 기준 Precision: 0.41
- 가입 클래스 기준 Recall: 0.60
- 가입 클래스 기준 F1-score: 0.49

이 결과는 전체 정확도뿐 아니라, 실제로 중요한 가입 고객을 일정 수준 이상 찾아내고 있다는 점에서 의미가 있습니다.

---

## 변수 중요도 분석

최종 RandomForest 모델의 변수 중요도를 확인한 결과, 가장 중요한 상위 변수는 다음과 같았습니다.

- nr.employed
- euribor3m
- emp.var.rate
- cons.conf.idx
- cons.price.idx

즉, 고객 개인의 단순 인구통계 정보보다도 당시의 경제 상황과 관련된 변수들이 정기예금 가입 여부에 더 큰 영향을 주는 것으로 나타났습니다.

이는 정기예금 가입이 개인 성향만의 문제가 아니라, 금리 수준과 경기 상황, 소비 심리 같은 거시경제적 환경에 민감하게 반응한다는 해석으로 이어질 수 있습니다.

---

## 추가 분석: 경제 지표 기반 변수 생성

변수 중요도 분석 결과를 바탕으로, 상위 경제 관련 변수를 결합해 경제 상황을 대표하는 새로운 변수 econ_score를 생성했습니다.

이후 econ_score를 기반으로 경제 상황을 구간화하여 가입률을 비교한 결과,

- 경제 상황이 좋지 않은 구간에서 정기예금 가입률이 더 높게 나타남
- 즉, 경기 둔화나 금리 변화와 같은 환경에서 고객이 보다 안정적인 금융 상품에 관심을 가질 가능성을 확인함

또한 econ_score를 실제 모델 입력 변수로 추가해 성능을 다시 비교한 결과,

- 기존 모델 ROC-AUC: 0.7921
- 추가 변수 반영 후 ROC-AUC: 0.7937

성능 향상 폭은 크지 않았지만, 데이터 분석을 통해 얻은 인사이트를 실제 피처 엔지니어링으로 연결했고, 그 결과가 예측 성능 개선으로도 이어졌다는 점이 의미 있었습니다.

## 이 프로젝트를 통해 갖추게 된 능력

### 1. 실무형 분류 문제를 구조적으로 해결하는 능력

단순히 모델을 돌리는 것이 아니라, 문제 정의부터 평가 지표 설정, 데이터 분할, 모델 비교, 최종 검증까지 전체 분류 프로젝트 흐름을 스스로 설계하고 수행할 수 있게 되었습니다.

### 2. 데이터 해석을 바탕으로 전처리를 설계하는 능력

age 로그 변환, campaign 구간화, pdays 재해석, duration 제거처럼 각 변수의 의미를 먼저 이해한 뒤 전처리 방식을 결정했습니다.  
이를 통해 정형 데이터 전처리를 기계적으로 하지 않고, 문제 맥락에 맞게 설계하는 역량을 키울 수 있었습니다.

### 3. 불균형 데이터 대응 능력

실제 분류 문제에서 자주 발생하는 클래스 불균형 문제를 SMOTETomek과 class_weight를 통해 다뤄보면서, 단순 정확도에 속지 않고 소수 클래스 예측 성능까지 고려하는 방법을 익힐 수 있었습니다.

### 4. 다양한 머신러닝 모델을 비교하고 선택하는 능력

로지스틱 회귀부터 RandomForest, XGBoost, CatBoost, LightGBM까지 여러 모델을 동일 기준으로 비교하면서, 어떤 모델이 이 데이터에 더 적합한지 근거를 가지고 설명할 수 있게 되었습니다.

### 5. 하이퍼파라미터 튜닝 능력

Optuna를 활용해 여러 트리 기반 모델을 튜닝하고 성능을 비교하면서, 모델 성능 향상을 위한 체계적인 탐색 과정을 경험했습니다.

### 6. 모델 결과를 비즈니스 인사이트로 연결하는 능력

변수 중요도를 해석하고, 경제 지표 기반 새로운 피처를 설계하며, 예측 결과를 단순 점수로 끝내지 않고 실제 금융 마케팅 전략과 연결해 해석하는 역량을 기를 수 있었습니다.

## 이 프로젝트에서 중요하게 본 점

이번 프로젝트에서 가장 중요하게 생각한 점은 예측 성능 자체보다, 왜 이런 결과가 나왔는지를 설명할 수 있어야 한다는 것이었습니다.

그래서 다음과 같은 원칙으로 프로젝트를 진행했습니다.

- 데이터 의미를 먼저 해석한 뒤 전처리하기
- 예측 시점에 사용할 수 없는 변수는 제거하기
- 불균형 데이터를 반드시 고려하기
- 하나의 모델만 쓰지 않고 여러 모델을 비교하기
- ROC-AUC 기준으로 최종 모델을 선정하기
- 변수 중요도를 통해 결과를 해석하고, 인사이트를 다시 피처 엔지니어링으로 연결하기

이러한 흐름은 향후 데이터 분석가나 AI 엔지니어로서 실무 문제를 다룰 때 매우 중요한 기반이 된다고 생각합니다.
